================================================================================
  BAB 6 — FORMAT DAN VALIDASI FIELD IDENTITAS
================================================================================

------------------------------------------------------------------------
6.1 ATURAN FORMAT NIK [WAJIB — validasi ketat]
------------------------------------------------------------------------
- Panjang         : 16 digit numerik, tidak boleh huruf
- Struktur        :
    Digit 1-2     : Kode provinsi (11–94)
    Digit 3-4     : Kode kabupaten/kota
    Digit 5-6     : Kode kecamatan
    Digit 7-12    : Tanggal lahir (DDMMYY)
                    Perempuan: tanggal + 40 (misal tgl 15 = 55)
    Digit 13-16   : Nomor urut pendaftaran di Dukcapil
- Karakter valid  : 0–9 saja
- Tidak boleh     : spasi, tanda hubung, tanda baca apapun

Regex validasi  : ^\d{16}$
Validasi kode provinsi: digit 1-2 harus antara 11 dan 94

------------------------------------------------------------------------
6.2 ATURAN FORMAT NISN [WAJIB untuk santri yang punya]
------------------------------------------------------------------------
- Panjang         : 10 digit numerik
- Dikeluarkan oleh: Pusdatin Kemdikbud (melalui VervalPD)
- Tidak bisa diubah setelah diterbitkan
- Jika belum punya: kosongkan, jangan isi "0000000000"
- Cara mendapatkan: melalui VervalPD atau koordinasi Kemenag kab/kota

Regex validasi  : ^\d{10}$

------------------------------------------------------------------------
6.3 ATURAN FORMAT NSP (Nomor Statistik Pesantren)
------------------------------------------------------------------------
- Panjang         : 12 digit numerik
- Dikeluarkan oleh: Ditjen Pendis Kemenag
- Ditetapkan melalui: SK Dirjen Pendis

------------------------------------------------------------------------
6.4 ATURAN FORMAT NPSN
------------------------------------------------------------------------
- Panjang         : 8 digit numerik
- Berlaku untuk   : Satuan pendidikan formal di bawah pesantren

------------------------------------------------------------------------
6.5 ATURAN FORMAT NAMA LENGKAP [WAJIB]
------------------------------------------------------------------------
- Minimal 3 karakter
- Maksimal 100 karakter
- Tidak boleh mencantumkan gelar:
  Dilarang: H, Hj, Haji, Hajah, Dr, Dra, Drs, Prof, Kyai, KH,
            Ust, Ustadz, S.Pd, M.Pd, S.Ag, M.Ag, S.H, M.H, dst.
- Tidak boleh menggunakan tanda:
  Dilarang: tanda petik ('), tanda kurung (), tanda apostrof
- Karakter yang diizinkan: huruf A-Z (a-z), spasi, tanda hubung (-)
- Gunakan nama sesuai dokumen resmi (KK/Akta Lahir/KTP)

------------------------------------------------------------------------
6.6 ATURAN FORMAT NOMOR HP
------------------------------------------------------------------------
- Diawali 08 atau +62
- Panjang: 10–14 digit
- Hanya angka (kecuali awalan +)
- Tidak mengandung spasi atau tanda hubung

------------------------------------------------------------------------
6.7 ATURAN FORMAT NOMOR KK (Kartu Keluarga)
------------------------------------------------------------------------
- Panjang: 16 digit numerik
- Format sama dengan NIK (sistem kependudukan Indonesia)

------------------------------------------------------------------------
6.8 ATURAN FORMAT NOMOR KIP / PIP
------------------------------------------------------------------------
- Panjang: bervariasi (biasanya 16 atau 19 digit)
- Hanya angka

================================================================================
  BAB 10 — CATATAN INTEGRASI DAN SINKRONISASI
================================================================================

------------------------------------------------------------------------
10.1 ALUR SINKRONISASI EMIS
------------------------------------------------------------------------
1. Input data di sistem internal pesantren (dengan kode referensi ini)
2. Validasi format lokal (NIK 16 digit, NISN 10 digit, dst.)
3. Dekripsi data terenkripsi di edge function (untuk field sensitif)
4. Pemetaan (mapping) kode internal ke kode EMIS resmi
5. Upload ke EMIS 4.0 via portal atau API yang tersedia
6. Verifikasi data oleh Kemenag Kab/Kota
7. BAP (Berita Acara Pendataan) ditandatangani dan diunggah kembali
8. Status data menjadi "Sudah BAP" — siap untuk:
   - Dana BOS
   - Penerbitan NISN
   - Rekomendasi beasiswa (PBSB, dll.)
   - Akreditasi lembaga

------------------------------------------------------------------------
10.2 VALIDASI KRUSIAL SEBELUM SINKRONISASI
------------------------------------------------------------------------
[ ] NIK santri tervalidasi Dukcapil (16 digit, format benar)
[ ] NISN terverifikasi di VervalPD (10 digit, bukan fiktif)
[ ] Status orang tua menggunakan nilai: HIDUP / MENINGGAL / TANPA KETERANGAN
[ ] Pekerjaan orang tua menggunakan kode 01–12 atau 99
[ ] Pendidikan orang tua menggunakan kode 01–11
[ ] Penghasilan menggunakan kode 1–6
[ ] Field ABK diisi (minimal kode 01 = Tidak Berkebutuhan Khusus)
[ ] NSP pesantren valid (12 digit) dan terdaftar di IJOP/SITREN
[ ] Santri mukim minimal 15 orang untuk lembaga terdaftar

------------------------------------------------------------------------
10.3 PEMUTAKHIRAN DATA (UPDATING SEMESTERAN)
------------------------------------------------------------------------
Semester Ganjil : Update dimulai 1 Juli, cut-off sekitar 30 Desember
Semester Genap  : Update dimulai 1 Januari, cut-off sekitar 30 Juni
Jadwal resmi    : Selalu cek surat edaran dari Ditjen Pendis terbaru
                  (biasanya diumumkan via Kanwil Kemenag masing-masing)

Data yang wajib diperbarui tiap semester:
- Status aktif/lulus/mutasi santri
- Rombongan belajar (rombel)
- Data GTK (guru/ustadz) yang masuk atau keluar
- Sarana dan prasarana lembaga
